# Corpus validation — v0.2.0 checks

Before releasing the three v0.2.0 checks (`hardport`, `maporder`, `eventuallyeffect`) we ran them
against the same four codebases as the [v0.1 corpus](2026-07-16-corpus-run.md) —
kubernetes/kubernetes, grafana/grafana, prometheus/prometheus, testcontainers-go — with the v0.1
checks disabled, and hand-triaged every finding.

## The gate did its job

The first pass surfaced 52 findings and **3 false positives**, all in the two hardest checks:

- `eventuallyeffect` flagged plain last-write-wins captures
  (`r, err = http.Get(...); return err == nil` with a guarded read after `Eventually`) — a
  deterministic, ubiquitous idiom. 2 dead-write FPs on prometheus, 24 of 25 sampled grafana
  findings were this idiom.
- `maporder` mis-attributed appends made inside a nested slice-range to the enclosing map-range
  when the accumulator was declared per-iteration (1 FP on grafana).

Both checks were narrowed before release:

- `eventuallyeffect` now flags only **count-dependent** effects — increments, compound
  assignments, self-appends, channel sends. Plain overwrites are silent by design.
- `maporder` ignores accumulators declared inside the range statement (per-iteration-fresh
  slices cannot depend on map iteration order).

## Final numbers (fixed binaries, same corpus)

| Check | kubernetes | grafana | prometheus | testcontainers-go | FP |
|---|---|---|---|---|---|
| `hardport` | 0 | 0 | 0 | 0 | 0 |
| `maporder` | 0 | 0 | 0 | 0 | 0 |
| `eventuallyeffect` | 0 | 0 | 0 | 0 | 0 |

Zero findings is a **precision** statement, not a recall one: these are mature codebases that
already hold ports open on `:0`, sort before asserting, and keep polling callbacks pure. We
verified each zero is real — the checks fire on synthetic reproductions, candidate call sites
were grep-audited (kubernetes has exactly zero hardcoded test binds; grafana binds
`127.0.0.1:0` throughout), and positive controls compiled in every scanned tree.

Notable true positives from the pre-fix pass that informed the final rules:

- grafana `pkg/infra/nats/integration_test.go` — a cross-poll settle baseline mutated inside an
  `Eventually` callback and asserted later (real premature-baseline flake). The narrowed check
  no longer reports it; a future rule for read-before-write cross-poll state is on the roadmap.
- prometheus keeps every bind behind `testutil.RandomUnprivilegedPort(t)` — the pattern
  `hardport` recommends, corpus-confirmed as the norm in healthy suites.

Known documented limits: `hardport` only sees constant addresses (a port routed through a
variable is silent); `eventuallyeffect` does not track pointer-mediated mutations
(`json.Unmarshal(b, &state)`).
